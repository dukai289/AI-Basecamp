# AI-Basecamp

一个用于沉淀 AI 相关内容的个人知识库。

这里不追求“什么都收藏”，而是把站点动态、AI 资讯和长期知识沉淀分开整理，方便后续检索、复盘和复用。

## 内容导航

| 板块 | 内容 |
| --- | --- |
| 动态 | 站点内容、结构和配置变更 |
| 资讯 | AI 行业动态、产品发布、重要论文、产业政策和事件跟踪 |
| 知识库 | 基础知识、教程指南、模型评测、工具框架、方法论和资源清单 |

## AI 知识库结构

| 板块 | 内容 |
| --- | --- |
| 知识体系 | AI 基础知识、概念解释、论文笔记和体系化资料 |
| 教程与指南 | 实操教程、上手指南、配置流程和案例步骤 |
| 模型评测 | 模型体验、能力评测、对比结论和使用建议 |
| 工具与框架 | Agent 开发、微调、测评、部署等工程实践 |
| 方法论 | 工作流、Prompt 策略和最佳实践 |
| 资源 | 课程、书籍、论文集合、GitHub 项目、数据集和工具站 |

## 收录原则

- 优先记录自己真正读过、试过、验证过的内容。
- 尽量保留原始来源链接，方便回溯。
- 对时效性强的内容标注日期，避免过期信息误导后续判断。
- 对评测类内容记录任务、样例、环境和限制条件，不只写主观感受。
- 对教程类内容写清楚前置条件和可复现步骤。
- 对资源清单类内容做主题分类，避免堆链接。

## 维护节奏

- 每日或每周：把新看到的重要 AI 资讯放入 `docs/news/`。
- 站点内容、结构和配置变更记录到 `docs/site-updates/index.md`。
- 试用新模型后：把体验和结论放入 `docs/model-reviews/`。
- 学到稳定知识后：整理到 `docs/knowledge/`。
- 跑通完整流程后：写到 `docs/guides-and-tutorials/`。
- 研究工具链后：沉淀到 `docs/tools-and-frameworks/`。
- 形成可复用做法后：归纳到 `docs/methods/`。
- 发现高质量资料后：补充到 `docs/resources/`。
